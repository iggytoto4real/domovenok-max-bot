## Режим «MAX без бэкенда» (max-fake)

Когда backend-core ещё не развёрнут, можно открывать мини-приложение **из MAX** с фейковыми данными:

- имя пользователя берётся из `initDataUnsafe`,
- список домовят — фейковый,
- API backend-core не вызывается.

### Как включить max-fake

Есть два способа:

1. **При сборке** — задать переменную окружения:

   ```bash
   cd mini-app
   set VITE_USE_FAKE_IN_MAX=true
   npm run build
   ```

   Либо в `.env`: `VITE_USE_FAKE_IN_MAX=true`.

2. **Через URL** — в настройках бота указать URL мини-приложения с параметром:

   `https://твой-фронт.vercel.app/?fake=1`

   Тогда даже без пересборки при открытии из MAX будет использоваться max-fake.

### Как это выглядит

В max-fake в шапке показывается имя из MAX, в теле — список фейковых домовят и кнопка «Добавить домовёнка».

Когда будет готов боевой бэкенд:

- убери `VITE_USE_FAKE_IN_MAX` из окружения/`.env` **или**
- убери параметр `fake` из URL мини-приложения

и задеплой фронт заново — включится полноценный prod-режим.
